# The big little details

We've talked about the theory behind good animations and how to actually code them. Now it's time to talk about the details that separate good animations from great animations. And we'll start with feeling.

## Feeling

Design has its rules for making a website appear luxurious, whimsical, or geeky. Using a serif font can make a website feel more premium, while using a monospace font indicates that the website is related to tech. Blue color is often used for trust, think of PayPal or Goldman Sachs. All of this helps us convey the feeling of the brand. But what about animations?

If we look at Stripe for example, we can see that their animations are pretty slow, they take their time. It feels premium and reliable — as if they are saying "we are not in a hurry, we are here for you". Strictly about the marketing website, not the dashboard.

If you are a young, forward-thinking startup, you'd probably want snappier animations. Think about Linear or Vercel — fast animations make these products feel innovative.

The animation style should match the brand. Slow and steady for premium, fast and snappy for innovative.

## Interruptions

A great animation handles interruptions gracefully. If the user hovers on and off quickly, the animation shouldn't glitch or jump. Spring animations handle this naturally because they maintain momentum.

## Crossfade and blur

Adding a subtle blur during transitions can mask imperfections and make state changes feel smoother. This is one of the most powerful "little details" that separates good from great.

## Stagger

Staggering animations — having elements animate in sequence with slight delays — adds rhythm and visual hierarchy. It guides the user's eye and makes the interface feel more alive.

## Direction awareness

Animations should be contextually aware. A menu opening from the left should animate from the left. A dropdown should animate from the trigger. Direction-aware animations feel more connected to the user's actions.

## Consistency

Consistency in your animation language is crucial. If buttons scale on press, all buttons should scale on press. If dropdowns use ease-out, all dropdowns should use ease-out. This makes the interface feel cohesive and predictable.
