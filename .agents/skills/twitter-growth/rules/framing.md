# Framing

## rules

- MUST describe what the USER does, not what the TOOL does — second person, imperative mood
- MUST name specific, recognizable pain points — not abstract solutions
- MUST be understandable in under 3 seconds without clicking through
- NEVER use emojis or hype adjectives ("blazing fast!", "revolutionary!") — correlates with lower performance

## user-centric vs feature-centric

**Broke out (400K+):**

- "Comment on the page → make a change to your app" → 1,189K views
- "Select an element → give it to AI" → 438K views
- "Scan your React codebase for anti-patterns" → 1,049K views

**Didn't break out (<200K):**

- "Automatically save clips when performance drops" → 36K views
- "Component tree view, smaller cleaner toolbar" → 41K views
- "Agent tests web app and fixes bugs" → 132K views

Pattern: breakout = "YOU do [action]". Non-breakout = "the tool does [action]".

## specificity

Concrete pain > abstract solution:

- "unnecessary useEffects, accessibility issues, prop drilling" → 1,049K views, 10,545 likes
- "scan your app for slow performance" → 572.8K views, 5,651 likes
- "save clips when performance drops" → 36K views, 621 likes

Template: `[YOU] do [simple action] → [impressive result]`
